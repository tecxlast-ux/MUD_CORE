# Visible Rules

PUNNARAJ prefers explicit rules over hidden assumptions.

Visible rules help humans and agents understand why a document exists, what layer it belongs to, how confident it is, and whether it can be safely reused.

## Working Rules

- Every published reference file declares layer, status, visibility, source layer, confidence, review date, owner, and tags.
- Public-readable does not mean canonical.
- Beta means reviewed enough to coordinate work, but still open to correction.
- Canonical status requires human validation.
- Derived notes must not pretend to be raw evidence.

Visible rules reduce drift and make review possible.
