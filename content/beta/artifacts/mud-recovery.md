# MUD Recovery Artifact

`mud` is a legacy failed repository recovered into a testbed and publication surface.

Its value is not that it began cleanly. Its value is that it can demonstrate recovery, bounded structure, and public-safe publishing after failure.

## Current Role

- Preserve legacy folders.
- Keep API routes and Pages Functions intact.
- Add reviewed reference layers without deleting history.
- Use build validation to prevent unsafe publication.

The recovered state should become more legible without erasing its origin.
